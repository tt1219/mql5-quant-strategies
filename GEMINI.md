# Gemini CLI Agent プロジェクト憲法 (GEMINI.md)

このファイルは、本プロジェクトにおける AI エージェントの行動指針と、開発・運用における絶対的なルールを定義します。

## 1. コミュニケーション・ルール
- **言語**: エージェントのすべての回答、解説、提案は、常に**日本語**で行うこと。
- **簡潔さ**: CLI環境に合わせ、高シグナルかつ簡潔な回答を心がけること。

## 2. MQL5 開発ルール
- **シンボル名（最重要）**: 
  - バックテストや取引において、シンボル名はブローカーの定義（例: `GOLD#`, `USDJPY#`）を正確に使用すること。末尾の `#` や記号を省略してはならない。
- **ディレクトリ構成**:
  - 新しいEAは `MQL5/Experts/Strategies/[EA名称]/[EA名称].mq5` のように、個別のフォルダを作成して配置すること。
  - プリセットファイルは `MQL5/Presets/` または EA 個別フォルダの `BacktestResults_Opt/` に整理すること。
- **実装スタンダード**:
  - 取引実行には `Trade/Trade.mqh` の `CTrade` クラスを使用すること。
  - `Include/AppCore/` 内の共通ライブラリ（TradeGuard, RiskManager, NewsFilter）を積極的に統合すること。
- **検証**:
  - コード変更後は必ず `MetaEditor64.exe` を使用してコンパイルを確認し、エラーを自己解決すること。
  - バックテストは `Shared/Automation/` 内のツールを使用して自動化を図ること。

## 3. Git & リポジトリ管理
- **コミット方針**:
  - ユーザーの明示的な指示がない限り、勝手に `git commit` や `git push` を行わないこと。
  - バックテスト結果（`Files/` フォルダ等）は Git 管理に含めず、`.gitignore` で適切に除外すること。
  - 重複したソースファイルや一時的なログファイル (`compile.log` 等) は随時削除し、リポジトリをクリーンに保つこと。
- **変更の評価**:
  - 既存のEA（`Rev_Bollinger` 等）に変更がある場合は、内容を精査し、最適化データなどの価値ある変更は適切に保存（コミット）すること。

## 4. プロジェクト特有の制約
- 現在のブランチ名や目的に基づき、スキャルピングEAであっても M5 や M15 などの適切な時間足での検証を優先すること。
