# Gemini CLI Agent プロジェクト憲法 (GEMINI.md)

このファイルは、本プロジェクトにおける AI エージェントの行動指針と、開発・運用における絶対的なルールを定義します。

## 1. コミュニケーション・ルール
- **言語**: エージェントのすべての回答、解説、提案は、常に**日本語**で行うこと。
- **簡潔さ**: CLI環境に合わせ、高シグナルかつ簡潔な回答を心がけること。

## 2. MQL5 開発ルール
- **シンボル名（最重要）**: 
  - バックテストや取引において、シンボル名はブローカーの定義（例: `GOLD#`, `USDJPY#`）を正確に使用すること。末尾の `#` や記号を省略してはならない。
- **ディレクトリ構成**:
  - 新しいEAは `MQL5/Experts/Strategies/[EA名称]/[EA名称].mq5` のように、個別のフォルダを作成して配置すること。
  - バージョン管理には**セマンティックバージョニング（MAJOR.MINOR.PATCH）**を適用すること。
- **実装スタンダード**:
  - 取引実行には `Trade/Trade.mqh` の `CTrade` クラスを使用し、`Include/AppCore/` 内の共通ライブラリを積極的に統合すること。
  - **クリーンビルドの徹底**: `.mq5` ファイルを変更した際は、必ず既存の `.ex5` ファイルを削除した上で再コンパイルを行い、変更が確実にバイナリに反映されたことを確認すること。

## 3. バックテスト評価基準 (3-Tier Standards)

エージェントは、開発したEAが以下の基準を満たしているかを厳格に検証しなければならない。

| ステージ | 期間 | 精度 | 目的 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| **Stage 1** | 1年間 | 1分足OHLC | 高速選別 | PF/取引数の足切り。スプレッド負荷 +0.5pips。 |
| **Stage 2** | 3年間 | 全ティック | 耐性検証 | リアルな変動への耐性。 |
| **Stage 3** | 10年間 | リアルティック | 最終確証 | 破綻リスクとOOS検証。 |

### Stage 1 合格目安（必須）
- **スキャルパー型** (高頻度): **PF 1.5以上** / トレード **120回以上** / MDD 15%以内
- **スナイパー型** (低頻度): **PF 1.7以上** / トレード **40-80回** / MDD 10%以内

## 4. 運用環境・検証条件
- **メイン口座**: XMTrading KIWAMI極 (KIWAMI Extreme)
- **レバレッジ**: 1:1,000
- **初期資金**: 100,000 JPY (または 1,000,000 JPY)
- **検証スプレッド**: 直近平均値に対し、安全策として **+0.5pips (50 points)** を加算して実行すること。

## 5. Git & リポジトリ管理
- **コミット方針**: ユーザーの明示的な指示がない限り、勝手に `git commit` を行わない。
- **クリーンアップ**: 重複したソースファイルや一時的なログファイル (`compile.log` 等) は随時削除し、リポジトリをクリーンに保つこと。
- **検証の停滞（キャッシュ）対策**:
  - ロジックの変更がバックテストに反映されない場合は、必ず対象の `.ex5` ファイルを物理削除してから再コンパイルを行うこと。
  - 自動コンパイルが失敗し続ける場合は、MetaEditor GUI を起動し、手動で「コンパイル」を実行して正常なバイナリを生成すること。
