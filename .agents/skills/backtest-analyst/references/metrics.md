# バックテスト指標詳細リファレンス

バックテストレポートに記載される各数値の意味と、EA評価における判定基準。

## 1. 収益性指標

### プロフィットファクター (PF)
- **計算式**: 総利益 / 総損失
- **判定**: 
  - `> 2.0`: 神レベル（過剰最適化を疑う）。
  - `1.5 - 2.0`: 非常に優秀。
  - `1.2 - 1.5`: 合格（実戦投入可能）。
  - `< 1.0`: 損失。

### 平均利益 / 平均損失
- **期待値計算**: (勝率 * 平均利益) - (負率 * 平均損失)
- **判定**: この値がスプレッド＋手数料（コスト）を大きく上回っている必要がある。

## 2. リスク指標

### 最大ドローダウン (Maximum Drawdown)
- **意味**: 資産の最大ピークからの最大下落幅。
- **判定**: 
  - `< 10%`: 極めて安全。
  - `10 - 20%`: 良好。
  - `> 30%`: 破綻リスク高。ロット調整が必要。

### リカバリーファクター (Recovery Factor)
- **計算式**: 総純損益 / 最大ドローダウン
- **意味**: ドローダウンをどれだけの速さで回復できるか。
- **判定**: `5.0` 以上を推奨。1年間のテストで `2.0` 未満は回復が遅すぎる。

## 3. 信頼性指標

### モデリング品質 (Modeling Quality)
- **判定**: 
  - `99%`: 全ティック（正確）。
  - `100%`: リアルティック（最も正確）。
  - `n/a` or `低い`: OHLCモデル（参考程度）。

### 取引数 (Total Trades)
- **意味**: 統計的な信頼性。
- **判定**: 1年間のテストで最低でも `100回` 以上は欲しい。回数が少なすぎると「たまたま勝った」可能性を排除できない。

## 4. 特殊なチェック項目

### シャープレシオ (Sharpe Ratio)
- **意味**: リスクに対するリターンの効率。
- **判定**: `0.2` 以上が望ましい。高いほど資産曲線が滑らか。

### Z-Score
- **意味**: 取引の連勝・連敗に相関があるか。
- **判定**: 負の値が大きいほど「負けた後に勝ちやすい（平均回帰）」、正の値が大きいほど「トレンドが出やすい」。
