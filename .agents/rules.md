# 言語ルール

- AI（Antigravity）からのすべての回答、思考（Thought）、ドキュメント、およびタスク管理ログは**日本語**で行う。
- テクニカルなファイル名や変数名、コードコメントなどは必要に応じて英語を使用できるが、説明やコミュニケーションは日本語を優先する。

## MQL5 開発・運用ナレッジ

### 1. 変数の制約と銘柄別上書き
- **制約**: `input` 変数はMQL5では定数（`const`）扱いであり、プログラム実行中に値を書き換えることができない。
- **対策**: `OnInit` 内で、銘柄（`_Symbol`）に応じてパラメータを分岐させたい場合は、必ず非定数の内部変数（`extBandsDev` 等）に値をコピーして使用すること。

### 2. メタエディタによる自動コンパイル
- **コマンド**: `metaeditor64.exe` を外部から呼び出す際は、PowerShellの引用符解釈の不具合を避けるため、`Start-Process` を使用するか、エスケープを適切に行う。
- **配置**: コンパイルを安定させるため、EA本体は `Experts` フォルダのルートに近い場所に配置するのが望ましい。

### 3. MT5 テンプレート (.tpl) と EA の自動アタッチ
- **構造**: 自動デプロイでEAをアタッチさせるには、テンプレート内の `<expert>` ブロックを `expertmode=1` と共に、通常は `<window>` ブロックの外側に配置する構造が安定する。

### 4. 開発環境のクリーン維持
- **リポジトリルート**: リポジトリのルートは `MQL5/` フォルダに設定されている。これにより、`Experts`, `Scripts`, `Indicators`, `Profiles`（テンプレート）などを一括管理する。
- **tempフォルダ**: 作業用の中間ファイルや一時的なバックアップは、常にルートの `temp/` フォルダに入れ、リポジトリ（Git）には含めない。
- **gitignore**: `.ex5` (バイナリ)、`.log` (ログ)、`.ini`、および `temp/` は常に除外対象とする。

### 5. MT5 ナビゲーターの表示ルール
- **ディレクトリ区分**: 以下のフォルダに配置されたファイルのみが、ナビゲーターの各カテゴリに表示される。
    - `MQL5\Experts\`: エキスパートアドバイザ (EA)
    - `MQL5\Scripts\`: スクリプト
    - `MQL5\Indicators\`: インディケータ
    - *注意*: スクリプトを `Experts` フォルダに置いても「スクリプト」欄には表示されない。
- **バイナリ必須**: ナビゲーターにはコンパイル済みの `.ex5` ファイルのみが表示される。`.mq5` ソースファイルのみの状態では表示されない（メタエディタ上には表示される）。

### 6. バックテストと集計の標準化
- **文字コードの罠**: MT5が出力するHTMLレポートは `UTF-16LE` 形式であり、改行コードの扱いも特殊である。安易に `grep` や PowerShell の標準出力で解析しようとすると文字化けやマッチング失敗を招く。
- **推奨エンジン**: レポートの自動集計には Node.js (fs.readFileSync(path, 'utf16le')) を使用することを強く推奨する。
- **パース戦略 (アンカー方式)**: レポートの構造は Build によって微妙に変化する。ラベル名（例: "総損益"）に頼るのではなく、「100%」（ヒストリー品質）などの不変のキーワードをアンカー（起点）として、そこからの相対的なインデックスで数値を特定する手法が最も堅牢である。
- **検証の徹底**: バックテスト実行後は、生成されたレポートの `TesterInputs` セクションを物理的にスキャンし、`InpUseNewsFilter=true` などの重要フラグが実際に適用されているかを必ず AI が自動確認すること。

### 7. バックテスト結果の管理と保存ルール
- **サマリの Git 管理**: バックテストの要約報告書（Markdown形式）は、各EAフォルダ内の `reports/` サブディレクトリに配置し、Git で管理対象とする。
- **生データの除外**: 詳細なHTMLレポート、CSV、および最適化ログ（`BacktestResults_Opt/` 等）は、Git 管理から除外（`.gitignore` で指定）する。

### 8. リポジトリ構成ルール (標準ディレクトリ構造)
- **`Experts/Strategies/` (EA戦略本体)**:
    - `Rev_[名称]/`: 逆張り(Reversion)型戦略のフォルダ
    - `Fol_[名称]/`: 順張り(Following)型戦略のフォルダ
    - 各戦略フォルダ内に `reports/` を配置し、最適化レポートを管理する。
- **`Include/AppCore/` (共通ライブラリ)**:
    - `NewsFilter.mqh`: ニュースフィルタ・停止処理
    - `RiskManager.mqh`: 資金管理・ロット計算
    - `TradeGuard.mqh`: スプレッド・スリッページ保護
- **`Shared/Automation/` (自動化・集計ツール)**:
    - `HyperTrading/automation/` にある汎用ツールを集約し、`env_config.ps1` で管理する。

### 9. バージョン管理とパラメータ同期
- **バージョン更新の義務**: EAのロジック変更、または最適化パラメータ（`InpAutoPreset` 等）を更新した際は、必ず `#property version` をインクリメントする。
- **共通ライブラリの利用**: 新規EA作成時は、可能な限り `AppCore` ライブラリを利用し、ニュースフィルタ等の基本機能を再利用し、コピー＆ペーストで再実装しないこと。
- **一貫性の維持**: EA内部のデフォルト値、プリセット値、およびデプロイ設定の3点は常に矛盾がないように管理する。
